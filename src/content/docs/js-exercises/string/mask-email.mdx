#### ðŸ’¡ Solution

```js
// example.js
function hideEmail(email) {
    const [localPart, domain] = email.split('@');
    const hiddenLocalPart = localPart.slice(0, 2) + '***';
    return `${hiddenLocalPart}@${domain}`;
}

// Test Data
console.log(hideEmail('example@gmail.com')); // "ex***@gmail.com"
console.log(hideEmail('john.doe@yahoo.com')); // jo***@yahoo.com
```

#### ðŸ“˜ Explanation

This function protects email addresses by hiding most of the first part (before the @) while keeping the domain visible.


1. **Splitting the Email:**
	- First, it splits the email into two parts using `split('@')`
	- Example: `'example@gmail.com'` becomes `['example', 'gmail.com']`

2. **Hiding the First Part:**
	- Takes the first part (before @) and keeps only the first 2 letters
	- Adds `***` to hide the rest
	- Example: `'example'` becomes `'ex***'`

3. **Keeping the Domain:**
	- The part after @ (domain) stays unchanged
	- Example: `'gmail.com'` remains `'gmail.com'`


4. **Combining the Parts:**
	- Puts them back together with @ in between"
	- Example: `'ex***' + '@' + 'gmail.com'` = `'ex***@gmail.com'`

- It's like partially covering an email address with masking tape
- Shows just enough to recognize who it might be
