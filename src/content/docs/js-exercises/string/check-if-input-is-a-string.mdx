```js
// example.js
function is_string(input) {
    return typeof input === 'string';
}

// Test Data
console.log(is_string('JavaScript')); // true
console.log(is_string([1, 2, 4, 0])); // false
```

#### ðŸ“˜ Explanation

1. `typeof input` checks what type of data the `input` is
2. `=== 'string'` compares it to see if it's exactly a text string type
3. **Returns:**
	- `true` if the input is a text string
	- `false` if it's anything else (number, array, object, etc.)


**Examples**

1. `is_string('JavaScript')` â†’ `true`

  - Because 'JavaScript' is a text string

2. `is_string([1, 2, 4, 0])` â†’ `false`

	- Because `[1, 2, 4, 0]` is an array, not a string

**Simple Real-life Comparison:**

It's like a bouncer at a club checking IDs:

- If your ID says "text string" â†’ You get in (`true`)
- If your ID says anything else (number, array, etc.) â†’ No entry (`false`)

**Key Notes:**

- Works for empty strings too (`''` is still a string)
- Only returns `true` for real strings, not things that look like strings
